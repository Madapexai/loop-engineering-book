使用本清单来检查你的Loop设计是否完整、可靠。

---

### 一、目标与范围

- [ ] **任务定义清晰**：输入是什么？输出是什么？用一句话能说清楚吗？

- [ ] **成功标准明确**：怎么算"做完了"？是可自动验证的吗？

- [ ] **范围边界清楚**：哪些事做，哪些事不做？有没有明确的"不做清单"？

- [ ] **风险评估到位**：最坏情况是什么？有没有预案？

- [ ] **投资回报合理**：花在搭建和维护上的时间，能通过节省的时间赚回来吗？

---

### 二、触发机制

- [ ] **触发方式明确**：时间触发？事件触发？条件触发？

- [ ] **去重机制**：同一个任务会不会被重复触发？

- [ ] **频率上限**：有没有最大触发频率限制？会不会被事件风暴冲垮？

- [ ] **优先级机制**：多个任务同时来，先处理哪个？

- [ ] **失败重试**：触发器本身失败了怎么办？

---

### 三、执行代理

- [ ] **模型选择合适**：任务的复杂度和模型的能力匹配吗？有没有大材小用或力不从心？

- [ ] **工具配置齐全**：代理有完成任务所需的所有工具吗？

- [ ] **权限设置合理**：遵循最小权限原则了吗？

- [ ] **Skill加载正确**：需要的项目知识和工作流程都加载了吗？

- [ ] **上下文充足**：代理能获取到完成任务所需的所有信息吗？

---

### 四、工作环境

- [ ] **隔离级别合适**：分支级？环境级？沙盒级？和任务风险匹配吗？

- [ ] **环境初始化可靠**：每次启动都能得到一致的干净环境吗？

- [ ] **资源限制到位**：CPU、内存、磁盘有没有限制？会不会耗尽系统资源？

- [ ] **清理机制完善**：任务结束后（无论成功失败）都会清理吗？

- [ ] **调试支持**：失败的任务能不能保留环境用于调试？

---

### 五、质量门禁

- [ ] **语法检查（G0）**：编译、lint、格式检查都有吗？

- [ ] **测试检查（G1）**：相关的测试都跑了吗？通过率多少？

- [ ] **静态分析（G2）**：安全扫描、复杂度检查做了吗？

- [ ] **AI审查（G3）**：有没有另一个代理来审查产出？

- [ ] **集成测试（G4）**：放到完整系统里能正常工作吗？

- [ ] **预演验证（G5）**：预发环境验证了吗？

- [ ] **人工确认（G6）**：高风险操作有人工确认环节吗？

- [ ] **门禁动态配置**：不同风险等级的任务走不同的门禁吗？

---

### 六、失败处理

- [ ] **退出条件明确**：Loop知道什么时候该停下来吗？

- [ ] **最大步数限制**：有没有硬性的步数/Token上限？

- [ ] **重试策略合理**：失败后怎么重试？重试几次？每次重试有什么不同？

- [ ] **失败模式识别**：能识别常见的失败模式（撞墙、失忆、跑飞…）吗？

- [ ] **升级机制完善**：搞不定的时候会通知人吗？通知谁？怎么通知？

- [ ] **边际收益检测**：连续多次没有实质改进的时候会停止吗？

---

### 七、记忆与学习

- [ ] **工作记忆**：任务过程中的关键信息有结构化记录吗？

- [ ] **长期记忆**：完成的任务会沉淀到长期记忆里吗？

- [ ] **Skill更新**：新的经验会更新到Skill库吗？

- [ ] **记忆质量控制**：错误的记忆会不会被标记或清理？

- [ ] **检索机制有效**：需要的时候能找到相关的记忆吗？

---

### 八、监控与度量

- [ ] **成功率**：知道多少任务成功、多少失败吗？

- [ ] **准确率**：成功的任务里，有多少是真的做对了？

- [ ] **平均耗时**：完成一个任务平均多久？

- [ ] **成本监控**：每个任务花多少Token/钱？有没有预算控制？

- [ ] **人工介入率**：多少任务需要人来处理？介入原因是什么？

- [ ] **审计日志**：所有操作都有完整的记录吗？

- [ ] **告警机制**：异常情况会触发告警吗？

---

### 九、安全与合规

- [ ] **最小权限**：代理只有完成任务必需的权限吗？

- [ ] **数据安全**：敏感数据有没有脱敏？会不会泄露？

- [ ] **高危操作确认**：删除、修改生产环境等操作有人工确认吗？

- [ ] **审计追踪**：所有操作都能追溯到具体哪个代理、什么时候、做了什么吗？

- [ ] **合规要求**：满足行业/公司的合规要求吗？

---

### 十、成本优化

- [ ] **任务路由**：不同任务用不同等级的模型吗？

- [ ] **上下文优化**：上下文是必要的吗？有没有冗余？

- [ ] **缓存机制**：相同的输入会复用结果吗？

- [ ] **重试成本**：每次重试都有新信息吗？还是在浪费Token？

- [ ] **预算控制**：有预算上限吗？超了会怎么样？
